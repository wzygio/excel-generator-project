# 知识提案：架构变更 · 2025-05-18

## 变更摘要
在 `yield_report` 领域内新增"数据获取层"完整实现，搭建从前端 UI 到后端基础设施的全栈脚手架，支持自然语言驱动的报表自动获取。

## 涉及文件清单
- [`src/yield_report/yield_report/application/orchestrator.py`](../../src/yield_report/yield_report/application/orchestrator.py): 新增 `DataAcquisitionOrchestrator` 编排器
- [`src/yield_report/yield_report/core/query_parser.py`](../../src/yield_report/yield_report/core/query_parser.py): 新增 `QueryParser` 自然语言解析核心
- [`src/yield_report/yield_report/infrastructure/finereport_client.py`](../../src/yield_report/yield_report/infrastructure/finereport_client.py): 新增 `FinereportClient` FineReport 下载器
- [`src/yield_report/yield_report/infrastructure/local_file_loader.py`](../../src/yield_report/yield_report/infrastructure/local_file_loader.py): 新增 `LocalFileLoader` 本地文件加载器
- [`src/yield_report/yield_report/infrastructure/product_models.py`](../../src/yield_report/yield_report/infrastructure/product_models.py): 新增产品型号提取工具
- [`app/main.py`](../../app/main.py): 新增 Tab3 "自然语言查询" 交互界面

## 建议更新的目标文件与插入位置

### A. 系统级架构变更 → 更新 [ARCHITECTURE.md](../../ARCHITECTURE.md)

#### 新增数据获取层描述

建议在 `ARCHITECTURE.md` 的 `## 3.3 领域模块划分` 章节中补充 `yield_report` 领域下的新模块：

**yield_report 领域（数据获取层）**：

| 模块 | 职责 | 文件 |
|------|------|------|
| `DataAcquisitionOrchestrator` | 应用层编排器：接收自然语言 → 调用 QueryParser 提取参数 → 调度 Infra 层获取数据 | `application/orchestrator.py` |
| `QueryParser` | 核心层：通过 LLMManager (JSON Mode) 将自然语言转为结构化 `ReportQueryRequest` | `core/query_parser.py` |
| `FinereportClient` | 基础设施：封装 FineReport 10.0 登录、Session 管理、参数提交、HTML 表格解析完整流水线 | `infrastructure/finereport_client.py` |
| `LocalFileLoader` | 基础设施：管理本地/网络路径文件就绪状态检查与复制 | `infrastructure/local_file_loader.py` |
| `ProductModelExtractor` | 基础设施：从 spotfire.xlsx 提取产品型号列表 | `infrastructure/product_models.py` |
